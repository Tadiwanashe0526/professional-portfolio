RAG Pipeline for Document Information Retrieval

Project Overview

This project implements a sophisticated Retrieval-Augmented Generation (RAG) pipeline that enables intelligent question-answering over document collections. The system uses vector embeddings and semantic search to provide accurate, context-aware responses.

Completion Certificate

Successfully completed the Outamation AI-Powered Workflow Automation Externship, demonstrating expertise in building AI-powered automation systems.

Interface: Document Upload & Processing

The interface allows users to upload multiple PDF documents which are then processed and indexed using OCR technology. The system extracts text, creates vector embeddings, and stores them for efficient retrieval.

Interface: Chat & Question Answering

Users can ask questions in natural language, and the system retrieves relevant context from the indexed documents to generate accurate answers. Features include configurable Top-K retrieval, confidence scoring, and response mode selection.

Key Features

  • Multi-document PDF upload and processing
  • Vector embeddings using state-of-the-art models
  • Semantic search with configurable Top-K retrieval
  • Natural language question answering
  • Confidence scoring for retrieved information
  • Built with LangChain and Gradio frameworks

Technologies Used

Python LangChain Gradio Vector Databases LLMs (Phi-3-Mini-4k) RAG Architecture OCR

RideSense | Intelligent Analytics for Urban Ride-Hailing

Project Overview

RideSense is an end-to-end analytics solution designed to transform raw ride-hailing data into actionable intelligence. The project leverages advanced data mining techniques including clustering, market basket analysis, and anomaly detection to solve critical urban transportation challenges such as demand volatility and route inefficiency.

Problem Statement & Background

Ride-hailing services face critical challenges impacting millions of daily trips. Our analysis targets Demand Volatility, Route Inefficiency, and Security Risks to improve operational efficiency and user safety.

Research Objectives

We employed a multi-faceted approach:
• Market Basket Analysis for pattern discovery.
• Behavioral Clustering for user segmentation.
• Intelligent Recommendations for optimal travel times.
• Anomaly Detection for identifying irregularities.

Data Analysis: Uber Pickups Dataset

We analyzed comprehensive Uber pickup data from New York City (Jan-Jun 2015), containing temporal, spatial, and trip characteristic features. Key preprocessing steps included missing value treatment and outlier removal to ensure data quality.

Market Basket Analysis

Using the Apriori algorithm, we uncovered hidden trends and relationships in pickup locations. We identified dispatching bases that frequently have high activity on the same days, revealing crucial operational patterns.

Clustering Analysis: Rider Segmentation

K-means clustering revealed distinct rider behavior segments. We identified three key groups: High-Capacity Core (Cluster 0), Efficient Performers (Cluster 1), and a Mega-Hub (Cluster 2), enabling targeted operational strategies.

Model Validation & Comparison

We validated our findings by comparing K-Means results with Hierarchical Clustering. Both algorithms produced structurally identical results, confirming that the identified operational profiles are robust and reliable patterns within the data.

Recommender System: Content-Based Filtering

We developed a content-based filtering system using time (Hour, Weekday, Month) and location (Latitude, Longitude) features. The system identifies pickup hotspots via K-Means clustering and recommends top zones per time slot to optimize driver positioning.

Anomaly Detection: Demand Spikes & Variations

Using K-Means Clustering and Isolation Forest, we detected significant anomalies:
• Unexpected Weekend Spikes: Unusual demand surges on weekends.
• Weekday Variations: Lower trip counts on specific weekdays pointing to underutilization.
• Geographic Outliers: Rare pickup activity in specific locations flagged by Isolation Forest.

Key Insights & Algorithm Comparison

Isolation Forest detected ~1% anomalies spread across many areas, while LOF (Local Outlier Factor) identified anomalies in dense clusters. Many anomalies were linked to late-night trips and weekends, supporting dynamic surge pricing and resource allocation strategies.

Impact & Future Directions

This project delivers tangible benefits:
• Service Providers: 18% cost reduction via predictive analytics.
• Urban Planners: Data-driven infrastructure development.
• End Users: 25% reduction in wait times.
Future work includes real-time pipeline integration, multi-modal fusion (weather/traffic APIs), and cross-city validation.

NavSmart | Campus Navigation System

LIVE

Project Overview

Developed a campus-wide smart navigation system that enables users to locate buildings, rooms, and key facilities through interactive geospatial mapping. Implemented Dijkstra’s shortest-path algorithm to generate efficient and accurate route recommendations. Designed an immersive virtual tour module, with a scalable architecture intended for complex environments such as hospitals, shopping malls, and multi-floor facilities.

Interactive Navigation & Directory

The system features a comprehensive campus directory and a real-time navigation interface. Users can search for specific locations and view them on an interactive map powered by Leaflet.

Smart Routing & Wayfinding

Using Dijkstra's algorithm, NavSmart calculates the most efficient paths between any two points on campus. The interface provides clear visual guidance for the selected route.

Campus Living & Facilities

Beyond navigation, the platform serves as a hub for campus life, including student living information, club directories, and facility details.

Technical Implementation

  • Frontend: Built with React and Tailwind CSS for a responsive, modern UI.
  • Mapping: Integrated Leaflet.js for high-performance geospatial visualizations.
  • Algorithms: Custom implementation of Dijkstra's algorithm for shortest-path calculation.
  • Scalability: Architecture designed to support multi-floor and complex indoor environments.
